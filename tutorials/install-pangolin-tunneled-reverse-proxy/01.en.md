## Introduction

Pangolin is a self-hosted reverse proxy that reaches private services over a WireGuard tunnel. Instead of opening ports on your home router or on an internal machine, you run a small client (Newt) next to the service, and it dials out to your public server. The server terminates HTTPS and forwards traffic down the tunnel. The result is similar to a Cloudflare Tunnel, except you own both ends.

This is a good fit for a small cloud server: it has a public IP and open ports 80/443, which is exactly what the private side lacks. At the end of this tutorial you will have a Pangolin dashboard reachable over HTTPS, a connected site, and one internal web app published on a public subdomain.

Pangolin is free and open-source software licensed under the GNU AGPL-3.0. This tutorial has been tested on a fresh Ubuntu 26.04 LTS server with Pangolin 1.19 and Traefik v3.6.

**Prerequisites**

* One server with a public IPv4 address. A fresh installation of Ubuntu 24.04 or 26.04 is fine.
* A domain name you control, with access to its DNS records
* The private service you want to publish — anything that speaks HTTP on a host you can run a client on

This tutorial uses the following example values:

* Username: `holu`
* IPv4 address: `10.0.0.1`
* Base domain: `example.com`
* Dashboard domain: `pangolin.example.com`

Replace them with your own values. All commands are run as `root` (or prefix them with `sudo`).

## Step 1 - Install Docker 

Pangolin runs on three containers, so you will need Docker and the Compose plugin.

On Ubuntu 26.04 the Docker APT repository did not yet list the `resolute` release codename when this was written, so the official convenience script is the reliable way to get a current build on both 24.04 and 26.04:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

Check that it worked:

```bash
docker --version
docker compose version
```

On the test server this returned `Docker version 29.5.3` and `Docker Compose version v5.1.4`. Any reasonably recent version is fine.

## Step 2 - Open the firewall ports

Pangolin requires four ports: two TCP for web traffic and two UDP for the WireGuard tunnel. To avoid locking yourself out, allow SSH first.

```bash
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 51820/udp
ufw allow 21820/udp
ufw enable
```

Port 80 is not optional. Pangolin issues certificates with the Let's Encrypt HTTP challenge, which only works if port 80 is reachable from the internet.

## Step 3 - Point your domain at the server

Pangolin gives every published service its own subdomain (`app.example.com`, `git.example.com`, and so on). So besides the dashboard record, you need a wildcard record. Create these DNS records:

| Type | Name              | Value         |
| ---- | ----------------- | ------------- |
| A    | `pangolin`        | `<server-ip>` |
| A    | `*`               | `<server-ip>` |

Add the certificate first, then start the stack — if the records don't resolve yet, Let's Encrypt will fail and Traefik will keep retrying. You can confirm a record has propagated with:

```bash
dig +short pangolin.example.com
```

It should print your server's IP before you continue.

## Step 4 - Create the configuration

Create the directory layout Pangolin expects:

```bash
mkdir -p /opt/pangolin/config/db /opt/pangolin/config/letsencrypt /opt/pangolin/config/traefik/logs
cd /opt/pangolin
```

### Step 4.1 - Application config

Generate a secret for the server:

```bash
openssl rand -hex 32
```

Create `config/config.yml` and paste the secret into the `secret` field. Replace the domains with yours:

```yaml
gerbil:
    start_port: 51820
    base_endpoint: "pangolin.example.com"

app:
    dashboard_url: "https://pangolin.example.com"
    log_level: "info"
    telemetry:
        anonymous_usage: true

domains:
    domain1:
        base_domain: "example.com"

server:
    secret: "paste-your-openssl-secret-here"
    cors:
        origins: ["https://pangolin.example.com"]
        methods: ["GET", "POST", "PUT", "DELETE", "PATCH"]
        allowed_headers: ["X-CSRF-Token", "Content-Type"]
        credentials: false

flags:
    require_email_verification: false
    disable_signup_without_invite: true
    disable_user_create_org: false
    allow_raw_resources: true
```

### Step 4.2 - Traefik config

Pangolin uses Traefik for TLS and routing. Create a file called `config/traefik/traefik_config.yml` and enter your Let's Encrypt email address.

```yaml
api:
  insecure: true
  dashboard: true

providers:
  http:
    endpoint: "http://pangolin:3001/api/v1/traefik-config"
    pollInterval: "5s"
  file:
    filename: "/etc/traefik/dynamic_config.yml"

experimental:
  plugins:
    badger:
      moduleName: "github.com/fosrl/badger"
      version: "v1.4.0"

log:
  level: "INFO"
  format: "common"

certificatesResolvers:
  letsencrypt:
    acme:
      httpChallenge:
        entryPoint: web
      email: "holu@example.com"
      storage: "/letsencrypt/acme.json"
      caServer: "https://acme-v02.api.letsencrypt.org/directory"

entryPoints:
  web:
    address: ":80"
  websecure:
    address: ":443"
    transport:
      respondingTimeouts:
        readTimeout: "30m"
    http:
      tls:
        certResolver: "letsencrypt"
      encodedCharacters:
        allowEncodedSlash: true
        allowEncodedQuestionMark: true

serversTransport:
  insecureSkipVerify: true

ping:
  entryPoint: "web"
```

Create `config/traefik/dynamic_config.yml`. Replace every `pangolin.example.com` with your dashboard domain:

```yaml
http:
  middlewares:
    badger:
      plugin:
        badger:
          disableForwardAuth: true
    redirect-to-https:
      redirectScheme:
        scheme: https

  routers:
    main-app-router-redirect:
      rule: "Host(`pangolin.example.com`)"
      service: next-service
      entryPoints:
        - web
      middlewares:
        - redirect-to-https
        - badger
    next-router:
      rule: "Host(`pangolin.example.com`) && !PathPrefix(`/api/v1`)"
      service: next-service
      entryPoints:
        - websecure
      middlewares:
        - badger
      tls:
        certResolver: letsencrypt
    api-router:
      rule: "Host(`pangolin.example.com`) && PathPrefix(`/api/v1`)"
      service: api-service
      entryPoints:
        - websecure
      middlewares:
        - badger
      tls:
        certResolver: letsencrypt
    ws-router:
      rule: "Host(`pangolin.example.com`)"
      service: api-service
      entryPoints:
        - websecure
      middlewares:
        - badger
      tls:
        certResolver: letsencrypt

  services:
    next-service:
      loadBalancer:
        servers:
          - url: "http://pangolin:3002"
    api-service:
      loadBalancer:
        servers:
          - url: "http://pangolin:3000"
```

### Step 4.3 - Compose file

Create `docker-compose.yml` in `/opt/pangolin`. Traefik shares Gerbil's network namespace, which is why only the Gerbil service publishes ports:

```yaml
name: pangolin
services:
  pangolin:
    image: docker.io/fosrl/pangolin:latest
    container_name: pangolin
    restart: unless-stopped
    volumes:
      - ./config:/app/config
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3001/api/v1/"]
      interval: "10s"
      timeout: "10s"
      retries: 15

  gerbil:
    image: docker.io/fosrl/gerbil:latest
    container_name: gerbil
    restart: unless-stopped
    depends_on:
      pangolin:
        condition: service_healthy
    command:
      - --reachableAt=http://gerbil:3004
      - --generateAndSaveKeyTo=/var/config/key
      - --remoteConfig=http://pangolin:3001/api/v1/
    volumes:
      - ./config/:/var/config
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    ports:
      - 51820:51820/udp
      - 21820:21820/udp
      - 443:443
      - 80:80

  traefik:
    image: docker.io/traefik:v3.6
    container_name: traefik
    restart: unless-stopped
    network_mode: service:gerbil
    depends_on:
      pangolin:
        condition: service_healthy
    command:
      - --configFile=/etc/traefik/traefik_config.yml
    volumes:
      - ./config/traefik:/etc/traefik:ro
      - ./config/letsencrypt:/letsencrypt
      - ./config/traefik/logs:/var/log/traefik

networks:
  default:
    driver: bridge
    name: pangolin
```

## Step 5 - Start Pangolin

```bash
docker compose up -d
```

The first run pulls three images and waits for the Pangolin container to report as healthy before Gerbil and Traefik are started. Check the state:

```bash
docker compose ps
```

All three containers should be `Up`, with `pangolin` showing `(healthy)`. If `traefik` keeps restarting, it almost always means the certificate request failed — recheck your DNS records and that port 80 is open.

## Step 6 - Create the admin account

Pangolin prints a one-time setup token to its log the first time it is started. Grab it!

```bash
docker compose logs pangolin | grep -A2 "SETUP TOKEN"
```

You'll see something like:

```
pangolin  | === SETUP TOKEN GENERATED ===
pangolin  | Token: 6s0h2j0zr1t7srbhaghfm7gntny07d8c
pangolin  | Use this token on the initial setup page
```

Open `https://pangolin.example.com/auth/initial-setup` in a browser. Paste the token, set the admin email and password, and create the account. You are redirected to the login page; sign in, and Pangolin walks you through creating your first organization.

## Step 7 - Connect a private site

After the organization is created, Pangolin opens the **Create Site** page. Leave the type on **Newt Site (recommended)** and give it a name. The page shows an ID, a secret, and a ready-made command for the host that runs your private service.

On that internal host, install the Newt client:

```bash
curl -fsSL https://static.pangolin.net/get-newt.sh | bash
```

Then connect it using the ID and secret from the dashboard:

```bash
sudo newt --id <site-id> --secret <site-secret> --endpoint https://pangolin.example.com
```

Newt dials out over UDP, so the private host needs no inbound ports. Back in the dashboard the site turns green once the tunnel is up.

## Step 8 - Publish a resource

In the dashboard go to **Resources** and add one. Pick the site you just connected, choose a subdomain (for example `app.example.com`), and point it at the internal address and port the service listens on, such as `http://localhost:8080`.

Save it, then open `https://app.example.com`. Traefik fetches a certificate for the new subdomain on the first request — this is why the wildcard DNS record from Step 3 matters — and your internal app is now served over HTTPS without a single inbound port on the private side.

## Conclusion

You set up Pangolin on a cloud server and published an internal service through an outbound WireGuard tunnel, with automatic HTTPS handled by Traefik. From here you can add more sites for other networks, put resources behind Pangolin's built-in authentication, or invite other users to the organization. The dashboard, Newt client, and resource list are all you need to manage from now on.
