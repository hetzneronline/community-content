## Introduction

FastAPI is a modern, high-performance Python web framework for building APIs. It is built on top of Starlette and Pydantic, supports asynchronous request handling natively, and automatically generates interactive API documentation. Running FastAPI in production requires a proper application server and a reverse proxy — in this tutorial, you will use **Gunicorn** with **Uvicorn workers** to serve the application and **Nginx** to handle incoming traffic and SSL termination.

By the end of this tutorial, your FastAPI application will be:
- Running persistently as a **systemd service**, automatically restarting on crashes and reboots
- Served by **Gunicorn** with multiple **Uvicorn workers** for concurrent request handling
- Accessible via **HTTPS** on your domain, with a free SSL certificate from Let's Encrypt

**Prerequisites**

- A Hetzner Cloud server running **Ubuntu 24.04** (CX22 or higher)
- A **domain name** pointing to your server's public IP address (an A record must already be set)
- A non-root user with `sudo` privileges, or access as root
- Basic familiarity with the Linux command line and Python

## Step 1 - Create a Dedicated User

Running web applications as root is a security risk. Create a dedicated system user for your FastAPI application:

```bash
sudo adduser fastapiuser
sudo usermod -aG sudo fastapiuser
```

Switch to the new user:

```bash
su - fastapiuser
```

## Step 2 - Install Python and pip

Ubuntu 24.04 ships with Python 3.12. Verify it is installed and install `pip` and the `venv` module:

```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv
```

Confirm the Python version:

```bash
python3 --version
```

## Step 3 - Set Up the Application

Create a directory for your FastAPI project and set up a Python virtual environment:

```bash
mkdir -p ~/myapi
cd ~/myapi
python3 -m venv venv
source venv/bin/activate
```

Install FastAPI, Uvicorn, and Gunicorn inside the virtual environment:

```bash
pip install fastapi uvicorn gunicorn
```

Create the main application file:

```bash
nano main.py
```

Add the following content:

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello from FastAPI on Hetzner!"}

@app.get("/health")
def health_check():
    return {"status": "ok"}
```

Save and close the file. Test that the application runs correctly:

```bash
uvicorn main:app --host 127.0.0.1 --port 8000
```

Open a second terminal and test the endpoint:

```bash
curl http://127.0.0.1:8000/
```

You should see:

```json
{"message": "Hello from FastAPI on Hetzner!"}
```

Press `Ctrl+C` to stop Uvicorn. In the next steps, Gunicorn and systemd will take over running the application.

## Step 4 - Configure Gunicorn

Gunicorn will manage multiple Uvicorn worker processes to handle concurrent requests efficiently. Create a Gunicorn configuration file:

```bash
nano ~/myapi/gunicorn.conf.py
```

Add the following content:

```python
# Bind to localhost on port 8000
bind = "127.0.0.1:8000"

# Number of worker processes
# A good starting point is (2 x number_of_cpu_cores) + 1
workers = 3

# Use Uvicorn workers for async support
worker_class = "uvicorn.workers.UvicornWorker"

# Logging
accesslog = "/var/log/gunicorn/access.log"
errorlog = "/var/log/gunicorn/error.log"
loglevel = "info"
```

Create the log directory:

```bash
sudo mkdir -p /var/log/gunicorn
sudo chown fastapiuser:fastapiuser /var/log/gunicorn
```

Test Gunicorn manually to confirm it starts correctly:

```bash
cd ~/myapi
source venv/bin/activate
gunicorn -c gunicorn.conf.py main:app
```

You should see Gunicorn start with 3 Uvicorn workers. Press `Ctrl+C` to stop it.

## Step 5 - Create a systemd Service

To run Gunicorn as a background service that starts automatically on boot, create a systemd unit file:

```bash
sudo nano /etc/systemd/system/myapi.service
```

Add the following content, replacing `fastapiuser` with your actual username if different:

```ini
[Unit]
Description=Gunicorn instance to serve FastAPI app
After=network.target

[Service]
User=fastapiuser
Group=fastapiuser
WorkingDirectory=/home/fastapiuser/myapi
Environment="PATH=/home/fastapiuser/myapi/venv/bin"
ExecStart=/home/fastapiuser/myapi/venv/bin/gunicorn -c /home/fastapiuser/myapi/gunicorn.conf.py main:app
ExecReload=/bin/kill -s HUP $MAINPID
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Save and close the file. Reload systemd, enable the service, and start it:

```bash
sudo systemctl daemon-reload
sudo systemctl enable myapi
sudo systemctl start myapi
```

Verify the service is running:

```bash
sudo systemctl status myapi
```

You should see `active (running)`. Check the application logs at any time with:

```bash
sudo journalctl -u myapi -f
```

## Step 6 - Install and Configure Nginx

Nginx will act as a reverse proxy, forwarding incoming HTTPS traffic to Gunicorn running on port 8000.

Install Nginx:

```bash
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

Create a new Nginx server block for your application:

```bash
sudo nano /etc/nginx/sites-available/myapi
```

Add the following configuration, replacing `api.example.com` with your actual domain:

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Enable the configuration and test for syntax errors:

```bash
sudo ln -s /etc/nginx/sites-available/myapi /etc/nginx/sites-enabled/
sudo nginx -t
```

You should see `syntax is ok`. Reload Nginx:

```bash
sudo nginx -s reload
```

Your API is now accessible over HTTP at `http://api.example.com`. In the next step, you will add HTTPS.

## Step 7 - Secure the Application with SSL

Install Certbot to obtain a free SSL certificate from Let's Encrypt:

```bash
sudo apt install -y certbot python3-certbot-nginx
```

Obtain and install the certificate:

```bash
sudo certbot --nginx -d api.example.com
```

Certbot will verify domain ownership, obtain the certificate, and automatically update your Nginx configuration to redirect HTTP to HTTPS. When prompted, enter your email address and agree to the terms of service.

Verify automatic renewal works:

```bash
sudo certbot renew --dry-run
```

Certbot installs a systemd timer that renews certificates automatically before they expire, so no manual action is needed.

## Step 8 - Configure the Firewall

Allow only the necessary ports:

```bash
sudo apt install -y ufw
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
```

Verify the rules:

```bash
sudo ufw status
```

## Step 9 - Verify the Deployment

Open your browser and navigate to:

```
https://api.example.com/
```

You should see:

```json
{"message": "Hello from FastAPI on Hetzner!"}
```

FastAPI also generates interactive API documentation automatically. Visit:

```
https://api.example.com/docs
```

You will see the Swagger UI with all your API endpoints listed and ready to test.

To check the health endpoint:

```bash
curl https://api.example.com/health
```

Expected response:

```json
{"status": "ok"}
```

To verify Gunicorn and the systemd service are running correctly:

```bash
sudo systemctl status myapi
tail -f /var/log/gunicorn/access.log
```

## Conclusion

You have successfully deployed a production-ready FastAPI application on your Hetzner server. Your setup includes:

- **FastAPI** served by **Gunicorn** with multiple **Uvicorn workers** for high concurrency
- A **systemd service** managing Gunicorn, with automatic restarts on failure and on server reboot
- **Nginx** as a reverse proxy, forwarding traffic from ports 80 and 443 to the application
- A **Let's Encrypt SSL certificate** installed via Certbot, with automatic renewal
- **UFW firewall** allowing only SSH and web traffic

From here, you can replace the sample `main.py` with your real FastAPI application, add database connections using SQLAlchemy or async drivers, and scale by increasing the number of Gunicorn workers in `gunicorn.conf.py`. For zero-downtime deployments, restart the service with:

```bash
sudo systemctl reload myapi
```
