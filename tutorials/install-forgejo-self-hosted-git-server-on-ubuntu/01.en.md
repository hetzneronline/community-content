## Introduction

Forgejo is a fully open-source, self-hosted Git service and a community-driven fork of Gitea. It provides a lightweight, fast, and feature-rich alternative to hosted platforms like GitHub or GitLab — allowing you to host your own repositories, manage teams, run CI/CD pipelines, and collaborate on code entirely on your own infrastructure. Forgejo is published under the MIT license and is actively maintained by the Codeberg community.

In this tutorial, you will install Forgejo on Ubuntu 24.04 using a binary release, configure it with a **PostgreSQL** database for production use, set up **Nginx** as a reverse proxy, and secure it with a free **Let's Encrypt SSL certificate**.

By the end of this tutorial, you will have a fully functional, private Git hosting platform accessible via HTTPS on your own domain.

**Prerequisites**

- A Hetzner Cloud server running **Ubuntu 24.04** (CX22 or higher; CX32 recommended for teams)
- A **domain name** pointing to your server's public IP address (an A record must already be set)
- A non-root user with `sudo` privileges, or access as root
- Basic familiarity with the Linux command line

## Step 1 - Create a Dedicated User

Forgejo should run as a dedicated non-root system user for security:

```bash
sudo adduser \
  --system \
  --shell /bin/bash \
  --gecos 'Forgejo' \
  --group \
  --disabled-password \
  --home /home/forgejo \
  forgejo
```

## Step 2 - Install and Configure PostgreSQL

Forgejo supports SQLite, MySQL, and PostgreSQL. PostgreSQL is recommended for production use as it offers better performance and reliability under concurrent load.

Install PostgreSQL:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable --now postgresql
```

Create a dedicated PostgreSQL user and database for Forgejo:

```bash
sudo -u postgres psql
```

Inside the PostgreSQL shell, run:

```sql
CREATE USER forgejo WITH PASSWORD 'StrongDBPassword123';
CREATE DATABASE forgejo OWNER forgejo ENCODING 'UTF8' LC_COLLATE 'en_US.UTF-8' LC_CTYPE 'en_US.UTF-8' TEMPLATE template0;
GRANT ALL PRIVILEGES ON DATABASE forgejo TO forgejo;
\q
```

Verify the connection works:

```bash
psql -U forgejo -h 127.0.0.1 -d forgejo -c "\l"
```

You should see the `forgejo` database listed.

## Step 3 - Download and Install Forgejo

Check the [Forgejo releases page](https://codeberg.org/forgejo/forgejo/releases) for the latest version. At the time of writing, the latest stable release is **7.0.0**. Replace the version number in the commands below if a newer version is available.

Create the required directories:

```bash
sudo mkdir -p /var/lib/forgejo/{custom,data,log}
sudo chown -R forgejo:forgejo /var/lib/forgejo
sudo chmod -R 750 /var/lib/forgejo
sudo mkdir -p /etc/forgejo
sudo chown root:forgejo /etc/forgejo
sudo chmod 770 /etc/forgejo
```

Download the Forgejo binary for Linux (amd64):

```bash
wget -O /tmp/forgejo https://codeberg.org/forgejo/forgejo/releases/download/v7.0.0/forgejo-7.0.0-linux-amd64
```

Verify the binary checksum by downloading and checking the SHA256 sum from the releases page:

```bash
wget -O /tmp/forgejo.sha256 https://codeberg.org/forgejo/forgejo/releases/download/v7.0.0/forgejo-7.0.0-linux-amd64.sha256
sha256sum -c /tmp/forgejo.sha256
```

You should see `OK`. Install the binary:

```bash
sudo mv /tmp/forgejo /usr/local/bin/forgejo
sudo chmod 755 /usr/local/bin/forgejo
```

Verify the installation:

```bash
forgejo --version
```

## Step 4 - Create the Forgejo Configuration File

Create the main Forgejo configuration file:

```bash
sudo nano /etc/forgejo/app.ini
```

Add the following content, replacing `git.example.com` with your actual domain and updating the database password:

```ini
APP_NAME = My Forgejo Instance
RUN_USER = forgejo
RUN_MODE = prod
WORK_PATH = /var/lib/forgejo

[server]
DOMAIN           = git.example.com
HTTP_PORT        = 3000
ROOT_URL         = https://git.example.com/
DISABLE_SSH      = false
SSH_PORT         = 22
LFS_START_SERVER = true
LFS_JWT_SECRET   =

[database]
DB_TYPE  = postgres
HOST     = 127.0.0.1:5432
NAME     = forgejo
USER     = forgejo
PASSWD   = StrongDBPassword123
SCHEMA   =
SSL_MODE = disable

[repository]
ROOT = /var/lib/forgejo/data/repositories

[log]
MODE      = console,file
LEVEL     = info
ROOT_PATH = /var/lib/forgejo/log

[security]
INSTALL_LOCK        = true
INTERNAL_TOKEN      =
SECRET_KEY          =

[service]
DISABLE_REGISTRATION              = false
REQUIRE_SIGNIN_VIEW               = false
REGISTER_EMAIL_CONFIRM            = false
ENABLE_NOTIFY_MAIL                = false
ALLOW_ONLY_EXTERNAL_REGISTRATION  = false
ENABLE_CAPTCHA                    = false
DEFAULT_KEEP_EMAIL_PRIVATE        = true
DEFAULT_ALLOW_CREATE_ORGANIZATION = true
DEFAULT_USER_IS_RESTRICTED        = false

[mailer]
ENABLED = false

[cache]
ADAPTER = memory

[session]
PROVIDER = file

[picture]
DISABLE_GRAVATAR        = false
ENABLE_FEDERATED_AVATAR = true

[openid]
ENABLE_OPENID_SIGNIN = false
ENABLE_OPENID_SIGNUP = false

[other]
SHOW_FOOTER_VERSION = false
```

Set the correct permissions on the configuration file:

```bash
sudo chown root:forgejo /etc/forgejo/app.ini
sudo chmod 640 /etc/forgejo/app.ini
```

Generate the required security tokens by running Forgejo's built-in generator:

```bash
forgejo generate secret INTERNAL_TOKEN
forgejo generate secret SECRET_KEY
forgejo generate secret LFS_JWT_SECRET
```

Copy the generated values and add them to `app.ini` under the corresponding keys in the `[security]` and `[server]` sections:

```bash
sudo nano /etc/forgejo/app.ini
```

Update the following lines with the generated values:

```ini
[server]
LFS_JWT_SECRET = <generated-lfs-jwt-secret>

[security]
INTERNAL_TOKEN = <generated-internal-token>
SECRET_KEY     = <generated-secret-key>
```

## Step 5 - Create a systemd Service

Create a systemd unit file to run Forgejo as a background service:

```bash
sudo nano /etc/systemd/system/forgejo.service
```

Add the following content:

```ini
[Unit]
Description=Forgejo - Beyond coding. We Forge.
Documentation=https://forgejo.org/docs/latest/
Wants=network.target
After=syslog.target network.target postgresql.service

[Service]
RestartSec=2s
Type=simple
User=forgejo
Group=forgejo
WorkingDirectory=/var/lib/forgejo
ExecStart=/usr/local/bin/forgejo web --config /etc/forgejo/app.ini
Restart=always
Environment=USER=forgejo HOME=/home/forgejo GITEA_WORK_DIR=/var/lib/forgejo

[Install]
WantedBy=multi-user.target
```

Reload systemd, enable, and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now forgejo
```

Check the service status:

```bash
sudo systemctl status forgejo
```

You should see `active (running)`. View the logs if needed:

```bash
sudo journalctl -u forgejo -f
```

## Step 6 - Install and Configure Nginx

Install Nginx:

```bash
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

Create a new Nginx server block for Forgejo:

```bash
sudo nano /etc/nginx/sites-available/forgejo
```

Add the following configuration, replacing `git.example.com` with your actual domain:

```nginx
server {
    listen 80;
    server_name git.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # Required for large file uploads and Git operations
        client_max_body_size 512m;
        proxy_read_timeout    3600;
        proxy_connect_timeout 3600;
        proxy_send_timeout    3600;
    }

    access_log /var/log/nginx/forgejo.access.log;
    error_log  /var/log/nginx/forgejo.error.log;
}
```

Enable the configuration and test it:

```bash
sudo ln -s /etc/nginx/sites-available/forgejo /etc/nginx/sites-enabled/
sudo nginx -t
sudo nginx -s reload
```

## Step 7 - Secure with SSL

Install Certbot:

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d git.example.com
```

Follow the prompts to enter your email and agree to the terms. Certbot will automatically configure HTTPS and set up HTTP-to-HTTPS redirection.

Verify automatic renewal:

```bash
sudo certbot renew --dry-run
```

## Step 8 - Configure the Firewall

```bash
sudo apt install -y ufw
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

## Step 9 - Initial Setup and First Login

Open your browser and navigate to:

```
https://git.example.com
```

You will see the Forgejo welcome page. Since `INSTALL_LOCK = true` is set in the configuration, the web installer is skipped and Forgejo is already configured.

Register your first administrator account by clicking **Register** on the homepage. The first user to register automatically becomes the administrator.

Fill in:
- **Username** — your admin username
- **Email** — your email address
- **Password** — a strong password

Click **Register Account**. You will be logged in and taken to the Forgejo dashboard.

### Step 9.1 - Create Your First Repository

Click the **+** button in the top right → **New Repository**. Give it a name, choose visibility (public or private), and click **Create Repository**.

You can now push code to your self-hosted Forgejo instance using standard Git commands:

```bash
git remote add origin https://git.example.com/yourusername/yourrepo.git
git push -u origin main
```

Or via SSH:

```bash
git remote add origin git@git.example.com:yourusername/yourrepo.git
git push -u origin main
```

### Step 9.2 - Disable Public Registration (Optional)

If you want to restrict your Forgejo instance to invited users only, edit the configuration file:

```bash
sudo nano /etc/forgejo/app.ini
```

Update the `[service]` section:

```ini
[service]
DISABLE_REGISTRATION = true
```

Restart Forgejo to apply the change:

```bash
sudo systemctl restart forgejo
```

## Conclusion

You now have a fully operational self-hosted Git platform running on your Hetzner server. Your setup includes:

- **Forgejo** running as a systemd service with automatic restarts
- **PostgreSQL** as the production-grade database backend
- **Nginx** as a reverse proxy with large file upload support for Git LFS
- **Let's Encrypt SSL** with automatic certificate renewal
- **UFW firewall** allowing only SSH and web traffic

From here, you can explore Forgejo's built-in features such as issue tracking, pull requests, project boards, webhooks, and Forgejo Actions — a GitHub Actions-compatible CI/CD system. You can also invite team members, create organizations, and mirror repositories from GitHub or GitLab for backup purposes.

For updates, download the new Forgejo binary from the releases page, replace the existing binary, and restart the service:

```bash
sudo systemctl stop forgejo
sudo mv /tmp/forgejo-new /usr/local/bin/forgejo
sudo chmod 755 /usr/local/bin/forgejo
sudo systemctl start forgejo
```
