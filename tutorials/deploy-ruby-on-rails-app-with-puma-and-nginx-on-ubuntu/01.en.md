## Introduction

Ruby on Rails is a full-stack web framework built on Ruby that emphasizes convention over configuration and developer productivity. In production, Rails applications are typically served by **Puma** — a fast, multi-threaded application server that ships as the default server in Rails 5 and above. Nginx sits in front of Puma as a reverse proxy, handling SSL termination, serving static assets directly, and forwarding dynamic requests to Puma.

In this tutorial, you will deploy a production-ready Ruby on Rails application on Ubuntu 24.04 using:
- **Ruby 3.3** installed via rbenv
- **Rails 7** as the web framework
- **Puma** as the application server
- **PostgreSQL** as the database
- **Nginx** as the reverse proxy
- **Let's Encrypt** for free SSL certificates via Certbot
- **systemd** to manage Puma as a persistent background service

**Prerequisites**

- A Hetzner Cloud server running **Ubuntu 24.04** (CX32 or higher recommended — Rails benefits from at least 4 GB RAM)
- A **domain name** pointing to your server's public IP address (an A record must already be set)
- A non-root user with `sudo` privileges, or access as root
- Basic familiarity with the Linux command line and Ruby

## Step 1 - Create a Dedicated User

Create a dedicated system user to own and run the Rails application:

```bash
sudo adduser railsuser
sudo usermod -aG sudo railsuser
su - railsuser
```

## Step 2 - Install System Dependencies

Install the required system packages for Ruby, Rails, and PostgreSQL:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl libssl-dev libreadline-dev zlib1g-dev \
    autoconf bison build-essential libyaml-dev libncurses5-dev \
    libffi-dev libgdbm-dev libpq-dev postgresql postgresql-contrib \
    nginx nodejs npm
```

Install Yarn, which is required by Rails for JavaScript asset management:

```bash
sudo npm install -g yarn
```

## Step 3 - Install Ruby with rbenv

rbenv is a lightweight Ruby version manager that allows you to install and switch between Ruby versions easily. Install rbenv and ruby-build:

```bash
git clone https://github.com/rbenv/rbenv.git ~/.rbenv
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(rbenv init -)"' >> ~/.bashrc
source ~/.bashrc

git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build
```

Install Ruby 3.3.0:

```bash
rbenv install 3.3.0
rbenv global 3.3.0
```

This may take a few minutes as Ruby is compiled from source. Verify the installation:

```bash
ruby --version
```

Install Bundler, the Ruby dependency manager:

```bash
gem install bundler
rbenv rehash
```

## Step 4 - Install and Configure PostgreSQL

Enable and start PostgreSQL:

```bash
sudo systemctl enable --now postgresql
```

Create a dedicated PostgreSQL user and database for your Rails application:

```bash
sudo -u postgres psql
```

Inside the PostgreSQL shell:

```sql
CREATE USER railsuser WITH PASSWORD 'StrongDBPassword123' CREATEDB;
CREATE DATABASE railsapp_production OWNER railsuser;
GRANT ALL PRIVILEGES ON DATABASE railsapp_production TO railsuser;
\q
```

## Step 5 - Deploy the Rails Application

### Step 5.1 - Create the Application Directory

```bash
sudo mkdir -p /var/www/myrailsapp
sudo chown railsuser:railsuser /var/www/myrailsapp
cd /var/www/myrailsapp
```

### Step 5.2 - Create a New Rails Application

For this tutorial, create a new Rails application. In a real scenario, replace this step with cloning your own repository:

```bash
gem install rails
rbenv rehash
rails new . --database=postgresql --skip-bundle
```

### Step 5.3 - Install Dependencies

```bash
bundle install
```

### Step 5.4 - Configure the Database

Open the database configuration file:

```bash
nano config/database.yml
```

Update the `production` section:

```yaml
production:
  adapter: postgresql
  encoding: unicode
  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
  database: railsapp_production
  username: railsuser
  password: <%= ENV["RAILS_DB_PASSWORD"] %>
  host: localhost
```

### Step 5.5 - Configure Environment Variables

Create an environment file to store sensitive configuration:

```bash
nano /var/www/myrailsapp/.env
```

Add the following:

```env
RAILS_ENV=production
RAILS_DB_PASSWORD=StrongDBPassword123
RAILS_MASTER_KEY=<your-master-key>
```

The `RAILS_MASTER_KEY` is found in `config/master.key`. View it with:

```bash
cat config/master.key
```

Set strict permissions on the environment file:

```bash
chmod 600 /var/www/myrailsapp/.env
```

### Step 5.6 - Set Up the Database and Assets

```bash
RAILS_ENV=production rails db:create db:migrate
RAILS_ENV=production rails assets:precompile
```

## Step 6 - Configure Puma

Puma ships with Rails and is configured via `config/puma.rb`. Open it:

```bash
nano config/puma.rb
```

Replace the contents with the following production configuration:

```ruby
# Number of threads per worker
max_threads_count = ENV.fetch("RAILS_MAX_THREADS") { 5 }
min_threads_count = ENV.fetch("RAILS_MIN_THREADS") { max_threads_count }
threads min_threads_count, max_threads_count

# Worker processes — set to number of CPU cores
workers ENV.fetch("WEB_CONCURRENCY") { 2 }

# Use a Unix socket for Nginx communication
bind "unix:///var/www/myrailsapp/tmp/puma.sock"

# PID and state files
pidfile ENV.fetch("PIDFILE") { "tmp/pids/server.pid" }
state_path "tmp/puma.state"

# Preload the app for faster worker forking
preload_app!

# Environment
rails_env = ENV.fetch("RAILS_ENV") { "production" }
environment rails_env

# Allow Puma to be restarted by `rails restart`
plugin :tmp_restart

on_worker_boot do
  ActiveRecord::Base.establish_connection if defined?(ActiveRecord)
end
```

Create the required directories:

```bash
mkdir -p /var/www/myrailsapp/tmp/pids
mkdir -p /var/www/myrailsapp/tmp/sockets
```

## Step 7 - Create a systemd Service for Puma

Create a systemd unit file to manage Puma:

```bash
sudo nano /etc/systemd/system/myrailsapp.service
```

Add the following content:

```ini
[Unit]
Description=Puma HTTP Server for Rails app
After=network.target postgresql.service

[Service]
Type=simple
User=railsuser
Group=railsuser
WorkingDirectory=/var/www/myrailsapp
EnvironmentFile=/var/www/myrailsapp/.env
ExecStart=/home/railsuser/.rbenv/shims/bundle exec puma -C /var/www/myrailsapp/config/puma.rb
ExecReload=/bin/kill -TSTP $MAINPID
Restart=always
RestartSec=5
KillMode=mixed
TimeoutStopSec=5

[Install]
WantedBy=multi-user.target
```

Reload systemd, enable, and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now myrailsapp
sudo systemctl status myrailsapp
```

You should see `active (running)`.

## Step 8 - Install and Configure Nginx

Create an Nginx server block for your Rails application:

```bash
sudo nano /etc/nginx/sites-available/myrailsapp
```

Add the following configuration, replacing `myrailsapp.example.com` with your actual domain:

```nginx
upstream puma {
    server unix:///var/www/myrailsapp/tmp/puma.sock;
}

server {
    listen 80;
    server_name myrailsapp.example.com;
    root /var/www/myrailsapp/public;

    # Serve static assets directly without hitting Puma
    location ^~ /assets/ {
        gzip_static on;
        expires max;
        add_header Cache-Control public;
    }

    # Serve uploaded files directly
    location ^~ /uploads/ {
        root /var/www/myrailsapp/public;
    }

    # Forward everything else to Puma
    location / {
        try_files $uri @puma;
    }

    location @puma {
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
        proxy_redirect off;
        proxy_pass http://puma;
    }

    # Error pages
    error_page 500 502 503 504 /500.html;
    client_max_body_size 100m;
    keepalive_timeout 10;

    access_log /var/log/nginx/myrailsapp.access.log;
    error_log  /var/log/nginx/myrailsapp.error.log;
}
```

Enable the configuration and test it:

```bash
sudo ln -s /etc/nginx/sites-available/myrailsapp /etc/nginx/sites-enabled/
sudo nginx -t
sudo nginx -s reload
```

## Step 9 - Secure the Application with SSL

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d myrailsapp.example.com
sudo certbot renew --dry-run
```

## Step 10 - Configure the Firewall

```bash
sudo apt install -y ufw
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

## Step 11 - Verify the Deployment

Open your browser and navigate to:

```
https://myrailsapp.example.com
```

You should see the Rails welcome page with a green padlock confirming HTTPS is active.

Check Puma is running correctly:

```bash
sudo systemctl status myrailsapp
sudo journalctl -u myrailsapp -f
```

Check Nginx logs:

```bash
tail -f /var/log/nginx/myrailsapp.access.log
tail -f /var/log/nginx/myrailsapp.error.log
```

## Conclusion

You have successfully deployed a production-ready Ruby on Rails application on your Hetzner server. Your complete stack includes:

- **Ruby 3.3** managed via rbenv for easy version switching
- **Puma** serving the Rails application via a Unix socket with multiple worker processes
- **systemd** managing Puma with automatic restarts on failure and on server reboot
- **Nginx** serving static assets directly and proxying dynamic requests to Puma
- **PostgreSQL** as the production database
- **Let's Encrypt SSL** with automatic certificate renewal
- **UFW firewall** allowing only SSH and web traffic

For future code deployments, update your application with the following workflow:

```bash
cd /var/www/myrailsapp
git pull origin main
bundle install
RAILS_ENV=production rails db:migrate
RAILS_ENV=production rails assets:precompile
sudo systemctl reload myrailsapp
```
