## Introduction

Beszel is a lightweight, self-hosted server monitor. It consists of two parts: a **hub** that provides a web dashboard, and an **agent** that you run on each machine you want to observe. The agent reports CPU, memory, disk, network, and per-container Docker metrics back to the hub, which stores the history and renders the graphs. This makes it a practical, low-overhead alternative to a full Prometheus and Grafana stack when you only need a clear overview of a small fleet.

In this tutorial, you install the hub on one server, create an admin account, and connect an agent so the dashboard shows live metrics. The agent runs on the same server as the hub here, but the same steps apply to any additional machine you want to monitor.

Beszel is free and open source (MIT). The steps were tested on Ubuntu 26.04 LTS with Beszel 0.18.7.

**Prerequisites**

* One server (a fresh Ubuntu 24.04 or 26.04 install works)
* Root access, or a user with `sudo`

This tutorial uses these example values:

* Username: `holu`
* IPv4 address: `10.0.0.1`

Replace them with your own values. Run all commands as `root`.

## Step 1 - Install Docker

Both the hub and the agent run as Docker containers. Install Docker with the official script:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

## Step 2 - Start the hub

Create a directory and a Compose file for the hub:

```bash
mkdir -p /opt/beszel
cd /opt/beszel
```

Create `/opt/beszel/docker-compose.yml` with the following content:

```yaml
services:
  beszel:
    image: henrygd/beszel:latest
    container_name: beszel
    restart: unless-stopped
    ports:
      - 8090:8090
    volumes:
      - ./beszel_data:/beszel_data
      - ./beszel_socket:/beszel_socket
```

Start the container:

```bash
docker compose up -d
```

The hub listens on port `8090`. Open that port in your firewall so you can reach the dashboard:

```bash
ufw allow 22/tcp
ufw allow 8090/tcp
ufw enable
```

## Step 3 - Create the admin account

Open `http://<your-server-ip>:8090` in a browser. On the first visit, Beszel prompts you to create an admin account. Enter an email address and a password, then submit. You land on an empty **All Systems** dashboard.

## Step 4 - Register a system

Click **Add System** in the top right. A dialog opens with several fields and two generated values:

- **Name** – a label for the machine, for example `web-1`
- **Host / IP** – the address of the machine running the agent. Use `localhost` when the agent runs on the same server as the hub.
- **Port** – leave the default `45876`
- **Public Key** – an `ssh-ed25519 …` key the hub generated for this system
- **Access Key** (token) – a short string the agent uses to register with the hub

Fill in the name and host, then copy the **Public Key** and the **Access Key** — you need both in the next step. Click **Add System** to save. The system appears in the list as not yet connected, because the agent is not running yet.

## Step 5 - Start the agent

Create a separate directory for the agent:

```bash
mkdir -p /opt/beszel-agent
cd /opt/beszel-agent
```

Create `/opt/beszel-agent/docker-compose.yml`. Replace `KEY` with the public key and `TOKEN` with the access key you copied from the dialog. Mounting the Docker socket gives the agent access to per-container statistics:

```yaml
services:
  beszel-agent:
    image: henrygd/beszel-agent:latest
    container_name: beszel-agent
    restart: unless-stopped
    network_mode: host
    environment:
      LISTEN: 45876
      HUB_URL: http://localhost:8090
      TOKEN: "paste-the-access-key-here"
      KEY: "ssh-ed25519 paste-the-public-key-here"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./beszel_agent_data:/var/lib/beszel-agent
```

Start the agent:

```bash
docker compose up -d
```

Check the logs to confirm it connected to the hub:

```bash
docker compose logs beszel-agent
```

A successful connection looks like this:

```
beszel-agent  | INFO Detected disk name=sda1 device=sda1 root=true
beszel-agent  | INFO Detected network interface name=eth0
beszel-agent  | INFO WebSocket connected host=localhost:8090
```

The agent initiates the connection to the hub. This means that when you add a remote machine later, no inbound port needs to be open on the agent side — only outbound access to the hub address is required.

## Step 6 - View the metrics

Go back to the dashboard and refresh the page. The system now shows up green with live readings for CPU, memory, disk, and network. Click the system name to open its detail page with history graphs. Switch to the **Containers** tab to see per-container stats from the Docker socket.

## Step 7 - Add more servers

To monitor another machine, repeat Steps 4 and 5 for it:

1. Click **Add System** in the hub to generate a fresh key and token for the new machine.
2. Install Docker on the new machine.
3. Deploy the agent there with `HUB_URL` pointing at your hub, for example `http://10.0.0.1:8090`.

Each agent gets its own credentials, and all systems appear side by side on the dashboard.

## Conclusion

You now have a working Beszel hub with one connected agent and live metrics visible on the dashboard. From here you can add the rest of your servers, configure threshold alerts for high CPU or low disk space in the system settings, or put the hub behind a reverse proxy with a domain name and HTTPS.

The agent is a single small process, so running it on every server adds negligible overhead.
