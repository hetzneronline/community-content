## Introduction

Beszel is a small, self-hosted server monitor. It has two parts: a **hub** with a web dashboard, and an **agent** that runs on each machine you want to watch. The agent reports CPU, memory, disk, network, and per-container Docker stats; the hub stores the history and draws the graphs. It is a much lighter option than a full Prometheus and Grafana stack when all you want is a clear overview of a handful of servers.

In this tutorial you install the hub on one server, create an admin account, and connect an agent so the dashboard shows live metrics. The agent here runs on the same server as the hub, but the exact same steps apply to any other machine you want to add.

Beszel is free and open source (MIT). This was tested on a fresh Ubuntu 26.04 LTS server with Beszel 0.18.7.

**Prerequisites**

* One server (a fresh Ubuntu 24.04 or 26.04 install is fine)
* Root access, or a user with `sudo`

This tutorial uses the following example values:

* Username: `holu`
* IPv4 address: `10.0.0.1`

Replace them with your own values. All commands are run as `root`.

## Step 1 - Install Docker

Both the hub and the agent run as containers. Install Docker with the official script, which covers Ubuntu 24.04 and 26.04:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

## Step 2 - Start the hub

Create a directory and a Compose file for the hub:

```bash
mkdir -p /opt/beszel
cd /opt/beszel
```

Put this in `/opt/beszel/docker-compose.yml`:

```yaml
services:
  beszel:
    image: henrygd/beszel:latest
    container_name: beszel
    restart: unless-stopped
    ports:
      - 8090:8090
    volumes:
      - ./beszel_data:/beszel_data
      - ./beszel_socket:/beszel_socket
```

Start it:

```bash
docker compose up -d
```

The dashboard listens on port `8090`. Open it in the firewall so you can reach it:

```bash
ufw allow 8090/tcp
```

## Step 3 - Create the admin account

Open `http://<your-server-ip>:8090` in a browser. On the first visit Beszel asks you to create an admin account — enter an email and password and submit. You land on an empty **All Systems** dashboard.

## Step 4 - Register a system

Click **Add System** in the top right. A dialog opens with a few fields and two generated values:

- **Name** – a label for the machine, e.g. `web-1`
- **Host / IP** – the address of the machine running the agent. For the agent on this same server, use `localhost`
- **Port** – leave the default `45876`
- **Public Key** – an `ssh-ed25519 ...` key the hub created for this system
- **Access Key** (token) – a short string used to register the agent

Fill in the name and host, then copy the **Public Key** and **Access Key** — you need both in the next step. Click **Add System** to save it. The system appears in the list, marked as not yet connected, because the agent isn't running yet.

## Step 5 - Start the agent

Create a separate directory for the agent so it has its own Compose file:

```bash
mkdir -p /opt/beszel-agent
cd /opt/beszel-agent
```

Put this in `/opt/beszel-agent/docker-compose.yml`. Replace `KEY` with the public key and `TOKEN` with the access key from the dialog. Mounting the Docker socket lets the agent report per-container statistics:

```yaml
services:
  beszel-agent:
    image: henrygd/beszel-agent:latest
    container_name: beszel-agent
    restart: unless-stopped
    network_mode: host
    environment:
      LISTEN: 45876
      HUB_URL: http://localhost:8090
      TOKEN: "paste-the-access-key-here"
      KEY: "ssh-ed25519 paste-the-public-key-here"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./beszel_agent_data:/var/lib/beszel-agent
```

Start the agent:

```bash
docker compose up -d
```

Check the log to confirm it reached the hub:

```bash
docker compose logs beszel-agent
```

You should see lines like these, ending with a successful connection:

```
beszel-agent  | INFO Detected disk name=sda1 device=sda1 root=true
beszel-agent  | INFO Detected network interface name=eth0
beszel-agent  | INFO WebSocket connected host=localhost:8090
```

The agent opens the connection to the hub itself, so when you later add a remote machine it only needs to reach the hub's address — no inbound port on the agent side.

## Step 6 - View the metrics

Go back to the dashboard and refresh. The system now shows up green with live readings — on the test server it reported processor 41.7%, memory 8.98%, storage 6.29%, a load average of 0.44, and the uptime. Click the system name to open its detail page with history graphs, and check the **Containers** page to see the per-container stats the Docker socket made available.

## Step 7 - Add more servers

To watch another machine, repeat Steps 4 and 5 on it: install Docker, click **Add System** in the hub to generate a fresh key and token for that machine, and run the agent there with `HUB_URL` pointing at your hub (for example `http://10.0.0.1:8090`). Each agent gets its own credentials, and they all appear side by side on the dashboard.

## Conclusion

You have a working Beszel hub with one connected agent and live metrics on the dashboard. From here you can add the rest of your fleet, set up alerts on thresholds like high CPU or low disk in the system settings, or put the hub behind a reverse proxy with a domain and HTTPS. It stays light: the agent is a single small process, so running it on every server costs almost nothing.
