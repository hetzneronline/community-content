## Introduction

Most AI tools are stateless — you open a chat, ask something, close it, and the context is gone. What I wanted was something different: an assistant that runs continuously on a server, remembers conversations, and is reachable through a messaging interface at any time.

This tutorial walks you through setting that up on a Hetzner Cloud VPS. By the end, you'll have a self-hosted AI agent that:

- Runs 24/7 as a system service on your server
- Uses [Ollama](https://ollama.com/) to run an open-source LLM locally — no OpenAI subscription needed
- Remembers conversation context within a session
- Responds through a Telegram bot you control

The total monthly cost is the price of a Hetzner CX22 server (~€4/month). Everything else is free and open source.

**Prerequisites**

- A [Hetzner Cloud](https://console.hetzner.cloud/) account
- An SSH key added to your Hetzner account ([guide](https://community.hetzner.com/tutorials/howto-ssh-key))
- A Telegram account and a bot token (free — instructions in Step 4)
- Basic Linux and Node.js familiarity

**Example terminology**

- Username: `holu`
- Server IP: `<10.0.0.1>`

---

## Step 1 - Create and Connect to the Server

In the [Hetzner Cloud Console](https://console.hetzner.cloud/), create a new server with the following settings:

- **Image:** Ubuntu 24.04
- **Type:** CX22 (2 vCPU, 4 GB RAM) — sufficient for a 7B parameter model
- **Location:** Pick the region closest to you
- **SSH Key:** Select your existing key

Once the server is running, connect to it:

```bash
ssh holu@<10.0.0.1>
```

Update the system before installing anything:

```bash
sudo apt update && sudo apt upgrade -y
```

---

## Step 2 - Install Ollama and Pull a Model

[Ollama](https://ollama.com/) lets you run large language models locally with a single command. It exposes a simple HTTP API on `localhost:11434` that we'll call from Node.js.

Install Ollama:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Pull a model. For a CX22 with 4 GB RAM, `llama3.2` (3B) runs well. If you have a larger server, `mistral` or `llama3.1:8b` work too:

```bash
ollama pull llama3.2
```

Test it quickly before moving on:

```bash
ollama run llama3.2 "Say hello in one sentence."
```

You should get a response within a few seconds. Once confirmed, exit the interactive session with `/bye`.

Verify the API is available:

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "What is 2+2?",
  "stream": false
}'
```

You should receive a JSON response with a `"response"` field. This is the endpoint our agent will use.

---

## Step 3 - Install Node.js

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
node -v  # should print v22.x.x
```

Create a directory for the agent:

```bash
mkdir ~/ai-agent && cd ~/ai-agent
npm init -y
```

Install the dependencies:

```bash
npm install node-telegram-bot-api node-fetch
```

- `node-telegram-bot-api` — handles the Telegram bot connection
- `node-fetch` — makes HTTP calls to the Ollama API

---

## Step 4 - Create a Telegram Bot

If you don't have a Telegram bot yet:

1. Open Telegram and search for `@BotFather`
2. Send `/newbot` and follow the prompts
3. Copy the **API token** (looks like `1234567890:ABCdef...`)

You'll also need your personal Telegram **chat ID**. Start a conversation with `@userinfobot` — it will reply with your numeric user ID.

Keep both values ready for the next step.

---

## Step 5 - Write the Agent

Create the main file:

```bash
nano ~/ai-agent/agent.js
```

Paste the following:

```javascript
import TelegramBot from 'node-telegram-bot-api';
import fetch from 'node-fetch';

const TELEGRAM_TOKEN = 'YOUR_TELEGRAM_BOT_TOKEN';
const ALLOWED_CHAT_ID = YOUR_CHAT_ID; // numeric, e.g. 123456789
const OLLAMA_URL = 'http://localhost:11434/api/chat';
const MODEL = 'llama3.2';

const SYSTEM_PROMPT = `You are a helpful personal assistant running on a private server.
You are concise, practical, and remember context within this conversation.`;

// Store conversation history per chat
const conversations = {};

const bot = new TelegramBot(TELEGRAM_TOKEN, { polling: true });

bot.on('message', async (msg) => {
  const chatId = msg.chat.id;

  if (chatId !== ALLOWED_CHAT_ID) {
    return bot.sendMessage(chatId, 'Unauthorized.');
  }

  const userText = msg.text;
  if (!userText) return;

  // Initialize history for this chat if needed
  if (!conversations[chatId]) {
    conversations[chatId] = [{ role: 'system', content: SYSTEM_PROMPT }];
  }

  conversations[chatId].push({ role: 'user', content: userText });

  // Keep context window manageable (last 20 messages + system prompt)
  if (conversations[chatId].length > 21) {
    conversations[chatId] = [
      conversations[chatId][0],
      ...conversations[chatId].slice(-20)
    ];
  }

  try {
    await bot.sendChatAction(chatId, 'typing');

    const response = await fetch(OLLAMA_URL, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        model: MODEL,
        messages: conversations[chatId],
        stream: false
      })
    });

    const data = await response.json();
    const reply = data.message?.content || 'No response.';

    conversations[chatId].push({ role: 'assistant', content: reply });

    await bot.sendMessage(chatId, reply);
  } catch (err) {
    console.error('Error:', err.message);
    await bot.sendMessage(chatId, 'Something went wrong. Check the server logs.');
  }
});

console.log('AI agent running...');
```

Replace `YOUR_TELEGRAM_BOT_TOKEN` and `YOUR_CHAT_ID` with your actual values.

Update `package.json` to enable ES modules:

```bash
npm pkg set type=module
```

Test that it works before setting up the service:

```bash
node agent.js
```

Send a message to your bot in Telegram. You should get a reply within a few seconds. Press `Ctrl+C` to stop.

## Step 6 - Run as a systemd Service

Running the agent manually is fine for testing, but a systemd service ensures it starts automatically on boot and restarts if it crashes.

Create the service file:

```bash
sudo nano /etc/systemd/system/ai-agent.service
```

Paste the following (replace `holu` with your actual username):

```ini
[Unit]
Description=Persistent AI Agent
After=network.target

[Service]
Type=simple
User=holu
WorkingDirectory=/home/holu/ai-agent
ExecStart=/usr/bin/node agent.js
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable ai-agent
sudo systemctl start ai-agent
```

Check that it's running:

```bash
sudo systemctl status ai-agent
```

You should see `active (running)`. To watch live logs:

```bash
journalctl -u ai-agent -f
```

Send another message to your Telegram bot to confirm it responds. Then reboot the server and verify it comes back automatically:

```bash
sudo reboot
```

After reconnecting, run `systemctl status ai-agent` — it should already be running without any manual intervention.

---

## Step 7 - Test and Verify Memory

Send a few messages to your bot to confirm conversation memory works:

```
You: My name is Alice and I work in finance.
Bot: Got it, Alice! How can I help you today?

You: What do you remember about me?
Bot: You mentioned your name is Alice and that you work in finance.
```

The agent retains context within a session. If you restart the service, conversation history resets — this is expected behavior for an in-memory store. If you need persistence across restarts, you can extend the agent to save and load conversation history from a JSON file.

---

## Conclusion

You now have a self-hosted AI agent running 24/7 on a Hetzner server. The key advantage of this setup over cloud AI products is full control: you choose the model, you own the data, and there are no per-request fees beyond your monthly server cost.

From here, you can extend this in many directions:

- **Swap the model** — try `mistral` or `llama3.1:8b` on a larger server for better quality
- **Add tools** — let the agent call external APIs (weather, calendar, home automation)
- **Persist memory** — save conversation history to disk so it survives restarts
- **Add a web interface** — expose the agent via a simple Express HTTP endpoint
- **Multi-user support** — extend the `ALLOWED_CHAT_ID` check to a list of allowed users
