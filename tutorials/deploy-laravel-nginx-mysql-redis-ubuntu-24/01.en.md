## Introduction

Laravel is one of the most popular PHP frameworks for building modern web applications. Deploying it correctly in a production environment requires careful setup of several components working together: **PHP-FPM** to run the application, **Nginx** as a reverse proxy and static file server, **MySQL** as the relational database, and **Redis** for high-speed caching, session storage, and queue management.

This tutorial walks you through every step needed to go from a bare Ubuntu 24.04 server to a fully running Laravel application — including user security, firewall setup, PHP configuration, Nginx virtual host, SSL with Let's Encrypt, and Redis integration.

**Prerequisites**

- A server running Ubuntu 24.04 (e.g. a Hetzner Cloud CX22 or larger)
- A non-root user with `sudo` privileges
- A domain name pointed at your server's IP address (required for SSL)
- Basic familiarity with the Linux command line

**What you will build:**

| Component | Role |
|-----------|------|
| PHP 8.3 + PHP-FPM | Runs the Laravel application |
| Nginx | Web server / reverse proxy |
| MySQL 8.0 | Relational database |
| Redis 7 | Cache, sessions, queue driver |
| Composer | PHP dependency manager |
| Let's Encrypt (Certbot) | Free TLS/SSL certificate |

## Step 1 — Initial Server Setup

Before installing any software, harden the server baseline.

### Step 1.1 — Connect and update

```bash
ssh root@<your-server-ip>
```

Update all packages:

```bash
apt update && apt upgrade -y
```

### Step 1.2 — Create a deploy user

Never run your application as root. Create a dedicated user:

```bash
adduser deployer
```

Add the user to the `sudo` group:

```bash
usermod -aG sudo deployer
```

Copy your SSH public key to the new user so you can log in without a password:

```bash
rsync --archive --chown=deployer:deployer ~/.ssh /home/deployer
```

From this point, log out and reconnect as `deployer`:

```bash
exit
ssh deployer@<your-server-ip>
```

### Step 1.3 — Configure the firewall

Ubuntu 24.04 ships with `ufw`. Enable it and open only the necessary ports:

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
```

> **Note:** `Nginx Full` opens both port 80 (HTTP) and 443 (HTTPS). After SSL is configured you can optionally restrict it to `Nginx HTTPS` only.

Check the status:

```bash
sudo ufw status
```

Expected output:

```
Status: active

To                         Action      From
--                         ------      ----
OpenSSH                    ALLOW       Anywhere
Nginx Full                 ALLOW       Anywhere
OpenSSH (v6)               ALLOW       Anywhere (v6)
Nginx Full (v6)            ALLOW       Anywhere (v6)
```

## Step 2 — Install PHP 8.3 and Required Extensions

Ubuntu 24.04 ships with PHP 8.3 in its official repositories.

### Step 2.1 — Install PHP-FPM and extensions

```bash
sudo apt install -y php8.3-fpm php8.3-cli php8.3-common \
    php8.3-mysql php8.3-redis php8.3-xml php8.3-mbstring \
    php8.3-curl php8.3-zip php8.3-bcmath php8.3-intl \
    php8.3-tokenizer php8.3-fileinfo php8.3-gd unzip curl git
```

Verify PHP is installed:

```bash
php -v
```

Expected output:

```
PHP 8.3.x (cli) (built: ...)
```

### Step 2.2 — Tune PHP-FPM settings

Open the main PHP-FPM configuration file:

```bash
sudo nano /etc/php/8.3/fpm/php.ini
```

Find and update the following values (use `Ctrl+W` in nano to search):

```ini
upload_max_filesize = 64M
post_max_size = 64M
memory_limit = 256M
max_execution_time = 60
```

Restart PHP-FPM to apply changes:

```bash
sudo systemctl restart php8.3-fpm
```

## Step 3 — Install and Configure MySQL

### Step 3.1 — Install MySQL

```bash
sudo apt install -y mysql-server
```

Run the security script to remove anonymous users and test databases:

```bash
sudo mysql_secure_installation
```

Answer the prompts:
- Set up the VALIDATE PASSWORD component: **Yes** (recommended)
- Password strength: **2** (strong)
- Remove anonymous users: **Yes**
- Disallow root login remotely: **Yes**
- Remove test database: **Yes**
- Reload privilege tables: **Yes**

### Step 3.2 — Create the database and user

Log in to MySQL as root:

```bash
sudo mysql
```

Inside the MySQL shell, run the following commands. Replace `your_db_password` with a strong password of your choice:

```sql
CREATE DATABASE laravel_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'laravel_user'@'localhost' IDENTIFIED BY 'your_db_password';
GRANT ALL PRIVILEGES ON laravel_db.* TO 'laravel_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

> **Tip:** Use a password manager (e.g. Bitwarden, KeePassXC) to generate and store your database password securely.

Verify the user was created correctly:

```bash
sudo mysql -u laravel_user -p laravel_db
```

Type your password when prompted. If you see the MySQL prompt, the setup is correct. Type `EXIT;` to leave.

## Step 4 — Install and Configure Redis

### Step 4.1 — Install Redis

```bash
sudo apt install -y redis-server
```

### Step 4.2 — Configure Redis for production

Open the Redis configuration file:

```bash
sudo nano /etc/redis/redis.conf
```

Make the following changes:

**Set a password** (required in production). Find the line:

```
# requirepass foobared
```

Uncomment it and set a strong password:

```
requirepass your_redis_password
```

**Bind only to localhost** (prevents remote access). Find the line:

```
bind 127.0.0.1 -::1
```

This is the default in Ubuntu 24.04 — confirm it is **not** commented out.

**Set supervised to systemd** so Redis integrates with the service manager. Find:

```
supervised no
```

Change it to:

```
supervised systemd
```

Save and exit (`Ctrl+X`, then `Y`, then `Enter`). Restart Redis:

```bash
sudo systemctl restart redis.service
```

Verify Redis is running and the password works:

```bash
redis-cli -a your_redis_password ping
```

Expected output:

```
PONG
```

## Step 5 — Install Composer

Composer is the dependency manager for PHP. Install it globally:

```bash
curl -sS https://getcomposer.org/installer -o /tmp/composer-setup.php
sudo php /tmp/composer-setup.php --install-dir=/usr/local/bin --filename=composer
```

Verify the installation:

```bash
composer --version
```

## Step 6 — Deploy the Laravel Application

### Step 6.1 — Create the web root directory

```bash
sudo mkdir -p /var/www/laravel
sudo chown -R deployer:deployer /var/www/laravel
```

### Step 6.2 — Clone or upload your application

If your code is in a Git repository:

```bash
git clone https://github.com/your-username/your-laravel-app.git /var/www/laravel
```

If you are starting a fresh Laravel project instead:

```bash
composer create-project laravel/laravel /var/www/laravel
```

### Step 6.3 — Install dependencies

```bash
cd /var/www/laravel
composer install --optimize-autoloader --no-dev
```

### Step 6.4 — Configure the environment file

Copy the example environment file:

```bash
cp .env.example .env
```

Generate the application key:

```bash
php artisan key:generate
```

Open the `.env` file:

```bash
nano .env
```

Update the following sections:

**Application settings:**

```dotenv
APP_NAME="My Laravel App"
APP_ENV=production
APP_KEY=base64:...  # already set by key:generate
APP_DEBUG=false
APP_URL=https://yourdomain.com
```

**Database settings:**

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=laravel_user
DB_PASSWORD=your_db_password
```

**Redis settings:**

```dotenv
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=your_redis_password
REDIS_PORT=6379
```

**Cache, session, and queue drivers** (switch them all to Redis):

```dotenv
CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis
```

### Step 6.5 — Set file permissions

Laravel needs write access to `storage` and `bootstrap/cache`:

```bash
sudo chown -R deployer:www-data /var/www/laravel
sudo chmod -R 755 /var/www/laravel
sudo chmod -R 775 /var/www/laravel/storage
sudo chmod -R 775 /var/www/laravel/bootstrap/cache
```

### Step 6.6 — Run database migrations

```bash
php artisan migrate --force
```

The `--force` flag is required in production environments to bypass the confirmation prompt.

### Step 6.7 — Optimize for production

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache
```

These commands compile your configuration, routes, views, and event listeners into cached files, significantly improving response time.

## Step 7 — Install and Configure Nginx

### Step 7.1 — Install Nginx

```bash
sudo apt install -y nginx
```

Check that Nginx is running:

```bash
sudo systemctl status nginx
```

### Step 7.2 — Create the Nginx server block

Remove the default site:

```bash
sudo rm /etc/nginx/sites-enabled/default
```

Create a new server block configuration file:

```bash
sudo nano /etc/nginx/sites-available/laravel
```

Paste the following configuration. Replace `yourdomain.com` with your actual domain name:

```nginx
server {
    listen 80;
    listen [::]:80;

    server_name yourdomain.com www.yourdomain.com;
    root /var/www/laravel/public;

    index index.php;

    # Logging
    access_log /var/log/nginx/laravel_access.log;
    error_log  /var/log/nginx/laravel_error.log;

    # Handle all requests through Laravel's front controller
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # Static file caching
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP-FPM processing
    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;

        # Security headers
        fastcgi_hide_header X-Powered-By;
    }

    # Block access to hidden files (e.g. .env, .git)
    location ~ /\. {
        deny all;
    }

    # Block direct access to vendor and node_modules
    location ~ ^/(vendor|node_modules) {
        deny all;
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";

    client_max_body_size 64M;
}
```

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/laravel /etc/nginx/sites-enabled/
```

Test the Nginx configuration for syntax errors:

```bash
sudo nginx -t
```

Expected output:

```
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
```

Reload Nginx:

```bash
sudo systemctl reload nginx
```

## Step 8 — Secure the Application with SSL (Let's Encrypt)

### Step 8.1 — Install Certbot

```bash
sudo apt install -y certbot python3-certbot-nginx
```

### Step 8.2 — Obtain and install the certificate

```bash
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

Follow the prompts:
- Enter your email address for renewal notices
- Agree to the Terms of Service: **Yes**
- Share your email with EFF (optional): your choice
- Redirect HTTP to HTTPS: **2** (Redirect — recommended)

Certbot will automatically update your Nginx configuration to serve traffic over HTTPS and redirect all HTTP requests.

### Step 8.3 — Verify automatic renewal

Certbot installs a systemd timer to renew certificates automatically. Test the renewal process with a dry run:

```bash
sudo certbot renew --dry-run
```

If no errors appear, automatic renewal is working correctly.

## Step 9 — Configure Laravel Queue Worker with Supervisor

If your application uses queued jobs (emails, notifications, etc.), you need a queue worker running persistently. Supervisor is a process control system that keeps the worker running even if it crashes.

### Step 9.1 — Install Supervisor

```bash
sudo apt install -y supervisor
```

### Step 9.2 — Create a Supervisor configuration

```bash
sudo nano /etc/supervisor/conf.d/laravel-worker.conf
```

Paste the following. Adjust the `numprocs` value based on your server capacity:

```ini
[program:laravel-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/laravel/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=deployer
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/laravel/storage/logs/worker.log
stopwaitsecs=3600
```

Reload Supervisor and start the workers:

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start laravel-worker:*
```

Check the status:

```bash
sudo supervisorctl status
```

Expected output:

```
laravel-worker:laravel-worker_00   RUNNING   pid 12345, uptime 0:00:05
laravel-worker:laravel-worker_01   RUNNING   pid 12346, uptime 0:00:05
```

## Step 10 — Configure the Laravel Scheduler

Laravel's task scheduler runs recurring commands (e.g. cleanup, reports). It relies on a single cron entry.

Open the crontab for the `deployer` user:

```bash
crontab -e
```

Add this line at the bottom:

```
* * * * * cd /var/www/laravel && php artisan schedule:run >> /dev/null 2>&1
```

Save and exit. The scheduler will now run every minute and execute any due tasks defined in `routes/console.php` (or `app/Console/Kernel.php` for older Laravel versions).

## Step 11 — Verify the Full Stack

### Step 11.1 — Check all services are running

```bash
sudo systemctl status nginx php8.3-fpm mysql redis supervisor
```

All five services should show `active (running)`.

### Step 11.2 — Test Redis connectivity from Laravel

```bash
cd /var/www/laravel
php artisan tinker
```

Inside tinker, run:

```php
Cache::put('test_key', 'hello', 60);
Cache::get('test_key');
```

Expected output:

```
=> "hello"
```

Type `exit` to leave tinker.

### Step 11.3 — Test the database connection

```bash
php artisan db:show
```

This displays your database connection details and confirms Laravel can reach MySQL.

### Step 11.4 — Visit your site

Open a browser and navigate to `https://yourdomain.com`. You should see your Laravel application served over HTTPS.

## Step 12 — Deploying Updates

Every time you push a new version of your application, run the following commands on the server:

```bash
cd /var/www/laravel

# Pull latest code
git pull origin main

# Install/update dependencies
composer install --optimize-autoloader --no-dev

# Run new migrations
php artisan migrate --force

# Clear and rebuild caches
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache

# Restart the queue workers to pick up code changes
php artisan queue:restart
```

> **Tip:** You can place these commands in a shell script (e.g. `deploy.sh`) and run it in one command on every deploy. For zero-downtime deployments, explore tools like **Deployer** or **Laravel Envoy**.

## Conclusion

You now have a fully production-ready Laravel application running on Ubuntu 24.04 with:

- **Nginx** serving the application with static file caching and security headers
- **PHP 8.3-FPM** processing all PHP requests efficiently
- **MySQL 8.0** storing your relational data
- **Redis 7** powering caching, sessions, and queue jobs
- **Let's Encrypt SSL** securing all traffic
- **Supervisor** keeping queue workers alive automatically
- **Laravel Scheduler** running periodic tasks via cron

From here, you can explore performance tuning (OPcache settings, Nginx worker processes, MySQL query cache) and monitoring tools such as Laravel Telescope or Prometheus + Grafana.
