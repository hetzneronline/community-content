## Introduction

Despite Hetzner does not offer an FreeBSD rescue system anymore it is possible to install and manage FreeBSD with OpenZFS from the Linux Rescue System on a dedicated server with UEFI boot. The installation will be done on a mirrored OpenZFS pool consisting of two drives.

**Prerequisites**

* Hetzner dedicated server with [UEFI boot](https://docs.hetzner.com/robot/dedicated-server/operating-systems/uefi/) enabled. To request UEFI boot, you need to file a free text support request for your [dedicated server](https://robot.hetzner.com/server) (Select the "Support"-Tab, and then the "Other" button)
* Server booted in Linux Rescue mode with working SSH access (root password or authorized key)
* Network configuration parameters - IP address, gateway, IPV6 address (any from the assigned subnet)

**Example terminology**

In our example we will be working with a system with two nvme drives `/dev/nvme0` and `/dev/nvme1`. To work with SATA drives you need to replace `/dev/nvme0n1` with `/dev/sda` and `/dev/nvme1n1` with `/dev/sdb`.

We will install FreeBSD 14.0-RELEASE in our example.

## Step 1 - Activate OpenZFS support on the rescue system

Type `zfs` in the command line and press "y". OpenZFS support is going to be installed.

```
zfs
```

### Step 2 - Download FreeBSD distribution files

Download `base.txz` and `kernel.txz` from a FreeBSD mirror.
```
curl -O http://ftp2.de.freebsd.org/pub/FreeBSD/releases/amd64/14.0-RELEASE/base.txz
curl -O http://ftp2.de.freebsd.org/pub/FreeBSD/releases/amd64/14.0-RELEASE/kernel.txz
```

## Step 3 - Create partitions
1. Run cfdisk with first drive
```
cfdisk /dev/nvme0n1
```
2. Select `gpt` as partition type
3. Press `N` (or select `New`) and type `100M` as partition size
4. Press `T` (or select `Type`) and select `EFI system` (very top)
5. Navigate with down arrow to "Free Space" and press `N` (or select `New`) and type e.g. `32G` as partition size for the swap partition
6. Press `T` (or select `Type`) and select `FreeBSD swap` (below all Linux partitions)
7. Navigate with down arrow to "Free Space" and press `N` (or select `New`) and choose any desired size for the OpenZFS partition (or use all space)
8. Press `T` (or select `Type`) and select `FreeBSD ZFS` (below all Linux partitions)
9. Press `Shift-W` (or select `Write`) to write out the partition table. Type `yes` to confirm the write.

10. Press `Q` (or select `Quit`)
11. Run cfdisk with second drive
```
root@rescue ~ # cfdisk /dev/nvme1n1
```
12. Repeat steps 3.2 to 3.11 with /dev/nvme1n1

### Step 4 - Create OpenZFS pool and root dataset

We are going to create a mirrored OpenZFS pool and on this pool the dataset `ROOT/FreeBSD`. We configure this dataset as the boot dataset.

```
zpool create -m none -O atime=off -O compression=on rpool mirror /dev/nvme0n1p3 /dev/nvme1n1p3
zfs create rpool/ROOT
zfs create -o mountpoint=legacy rpool/ROOT/FreeBSD
zpool set bootfs=rpool/ROOT/FreeBSD rpool
```

If you want to install FreeBSD 13 instead of 14, you need to use the flag `-o compatibility=openzfs-2.1-freebsd` with `zpool create`.

### Step 5 - Mount OpenZFS dataset and extract freebsd distribution

```python
mount -t zfs rpool/ROOT/FreeBSD /mnt
bsdtar -x -C /mnt -f base.txz
bsdtar -x -C /mnt -f kernel.txz
```

If `bsdtar` is not installed, install with `apt -y install libarchive-tools`.

## Step 4 - Format EFI partitions
Format the EFI partitions and copy FreeBSD EFI boot loader.
```
mkfs.fat -s 1 -F 32 /dev/nvme0n1p1
mkfs.fat -s 1 -F 32 /dev/nvme1n1p1
mkdir -p /efi /efi2
mount -t vfat /dev/nvme0n1p1 /efi
mount -t vfat /dev/nvme1n1p1 /efi2
mkdir -p /efi/EFI/BOOT /efi2/EFI/BOOT
cp /mnt/boot/loader.efi /efi/EFI/BOOT/BOOTX64.efi
cp /mnt/boot/loader.efi /efi2/EFI/BOOT/BOOTX64.efi
```

## Step 5 - Configure FreeBSD installation

### Step 5.1 Create /mnt/boot/loader.conf
Create the file `/mnt/boot/loader.conf`, with the following contents:
```
cat << EOF > /mnt/boot/loader.conf
zfs_load="YES"
vfs.root.mountfrom="zfs:rpool/ROOT/FreeBSD"
autoboot_delay="3"
hw.nvme.use_nvd="0"
kern.geom.label.disk_ident.enable="0"
EOF
```

### Step 5.2 Create /etc/rc.d/autodhcp
If we do not know the name of the network interface yet, we need to configure all interfaces to `DHCP`. If you know the network interface name (e.g. `igb0`), you can skip this step and step 5.3 and directly use `rc.conf` code from Step 6.

Create the file `/mnt/etc/rc.d/autodhcp` with the following contents:
```
cat << EOF > /mnt/etc/rc.d/autodhcp
#!/bin/sh

# PROVIDE: autodhcp
# BEFORE: NETWORKING netif routing hostname
# REQUIRE: mountcritlocal mdinit
# KEYWORD: FreeBSD

. /etc/rc.subr

name=autodhcp
rcvar=autodhcp_enable

load_rc_config $name

: \${autodhcp_enable:="NO"}

start_cmd="autodhcp_start"
stop_cmd=":"

autodhcp_start()
{
        _dif=\$(/sbin/ifconfig -l | /usr/bin/sed -E 's/lo[0-9]+//g')
        for i in \$_dif; do
                echo "ifconfig_\$i=\"DHCP\"" >> /etc/rc.conf.d/network
        done
}

load_rc_config \$name
run_rc_command "\$1"
EOF
```

Now make the file executable:
```
chmod +x /mnt/etc/rc.d/autodhcp
```

### Step 5.3 Edit /mnt/etc/rc.conf
We need to set the hostname, enable sshd and our newly created autodhcpd script.

Create the file `/mnt/etc/rc.conf` with the following contents:
```
cat << EOF > /mnt/etc/rc.conf
hostname="myhost.mydomain"
sshd_enable="YES"
autodhcp_enable="YES"
EOF
```

### Step 5.4 Configure root SSH key
As we cannot create the password database from Linux, we have to login with a SSH key.

```
mkdir -p /mnt/root/.ssh
chmod 0400 /mnt/root/.ssh
```

Now you have to copy your public key into `/mnt/root/.ssh/authorized_keys`

### Step 5.5 Enable root login via SSH
```
echo "PermitRootLogin without-password" >> /mnt/etc/ssh/sshd_config
```

### Step 5.6
Reboot
```
reboot
```

## Step 6 - Post-Configuration
SSH as root to the system using its IPv4 address and the correct SSH key.

### Step 6.1 - Delete /etc/rc.conf.d/network that was generated by autodhcp
```
rm /etc/rc.conf.d/network
```

### Step 6.2 - Reconfigure /etc/rc.conf

From the netstat output we can see the default gateway and the network interface:
```
root@myhost:~ # netstat -rn | grep default
default            192.168.0.1         UGS        igb0
```

In this example, the network interface is igb0 and default gateway is 192.168.0.1.

Now we must rewrite `/etc/rc.conf`. Please use your interface instead of `igb0` (don't forget to replace in ipv6_defaultrouter), your IPv4 address instead of `192.168.0.35`, the gateway you discovered instead of `192.168.0.1` and your IPv6 address instead of `2a01:4f8:0:0::2` (you should see the IPv6 address scope in your Hetzner Robot server details).
```
hostname="myhost.mydomain"
sshd_enable="YES"
ifconfig_igb0="inet 192.168.0.35/32"
ifconfig_igb0_ipv6="inet6 2a01:4f8:0:0::2"
gateway_if="igb0"
gateway_ip4="192.168.0.1"
static_routes="gateway default"
route_gateway="-host $gateway_ip4 -interface $gateway_if"
route_default="default $gateway_ip4"
ipv6_defaultrouter="fe80::1%igb0"
```

### Step 6.3 Set Root Password
If you want to set the root password, you can do it now with
the passwd command:
```
passwd
```

### OPTIONAL: Step 7: Setup mirrored swap device
You can now setup a gmirror-ed swap drive:
```
echo geom_mirror_load=\"YES\" >> /boot/loader.conf
gmirror label -F swap0 nda0p2 nda1p2
echo /dev/mirror/swap0 none swap sw 0 0 >> /etc/fstab
swapon -a
```
