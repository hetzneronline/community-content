## Introduction

Node.js is a popular JavaScript runtime for building fast, scalable server-side applications. Running a Node.js application in production requires more than just executing `node app.js` — you need a process manager to keep the app alive after crashes and reboots, and a reverse proxy to handle SSL termination and serve traffic on standard ports 80 and 443.

In this tutorial, you will deploy a Node.js application on Ubuntu 24.04 using **PM2** as the process manager and **Nginx** as the reverse proxy. You will also secure the application with a free SSL certificate from Let's Encrypt using Certbot.

By the end of this tutorial, your Node.js application will be:
- Running persistently in the background, managed by PM2
- Automatically restarted on crashes or server reboots
- Accessible via HTTPS on your domain

**Prerequisites**

- A Hetzner Cloud server running **Ubuntu 24.04** (CX22 or higher)
- A **domain name** pointing to your server's public IP address (an A record must already be set)
- A non-root user with `sudo` privileges, or access as root
- Basic familiarity with the Linux command line

## Step 1 - Create a Dedicated User

Running applications as root is a security risk. Create a dedicated system user for your Node.js application:

```bash
sudo adduser nodeuser
sudo usermod -aG sudo nodeuser
```

Switch to the new user for the rest of the setup:

```bash
su - nodeuser
```

## Step 2 - Install Node.js

Ubuntu 24.04's default repositories include an older version of Node.js. To get the latest LTS version, use the NodeSource repository.

First, install the required dependencies:

```bash
sudo apt update
sudo apt install -y curl
```

Add the NodeSource repository for Node.js 22 LTS and install it:

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

Verify the installation:

```bash
node --version
npm --version
```

You should see the installed versions of both Node.js and npm.

## Step 3 - Deploy Your Application

For this tutorial, you will create a simple Node.js HTTP server to demonstrate the full deployment workflow. In a real scenario, you would replace this with your own application code.

Create a directory for your application:

```bash
mkdir -p ~/myapp
cd ~/myapp
```

Initialize a new Node.js project:

```bash
npm init -y
```

Create the main application file:

```bash
nano app.js
```

Add the following content:

```javascript
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello from my Node.js app on Hetzner!\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Save and close the file. Test that the application runs correctly:

```bash
node app.js
```

You should see `Server running at http://127.0.0.1:3000/`. Press `Ctrl+C` to stop it. In the next step, PM2 will take over running the application.

## Step 4 - Install and Configure PM2

PM2 is a production process manager for Node.js. It keeps your application running in the background, restarts it automatically on crashes, and can launch it on system boot.

Install PM2 globally using npm:

```bash
sudo npm install -g pm2
```

Start your application with PM2:

```bash
cd ~/myapp
pm2 start app.js --name "myapp"
```

Check the status of your running application:

```bash
pm2 status
```

You should see `myapp` listed with a status of `online`.

### Step 4.1 - Configure PM2 to Start on Boot

Generate and configure the startup script so PM2 (and your app) automatically starts after a server reboot:

```bash
pm2 startup
```

PM2 will print a command that you need to run with `sudo`. Copy and run that command. It will look similar to:

```bash
sudo env PATH=$PATH:/usr/bin /usr/lib/node_modules/pm2/bin/pm2 startup systemd -u nodeuser --hp /home/nodeuser
```

After running it, save the current PM2 process list so it is restored on reboot:

```bash
pm2 save
```

### Step 4.2 - Useful PM2 Commands

Here are some commands you will use frequently to manage your application:

| Command | Description |
|---|---|
| `pm2 status` | Show all running processes |
| `pm2 logs myapp` | Show application logs |
| `pm2 restart myapp` | Restart the application |
| `pm2 stop myapp` | Stop the application |
| `pm2 reload myapp` | Reload with zero downtime |
| `pm2 delete myapp` | Remove the application from PM2 |

## Step 5 - Install and Configure Nginx

Nginx will act as a reverse proxy, forwarding incoming HTTP and HTTPS traffic to your Node.js application running on port 3000.

Install Nginx:

```bash
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

Verify it is running:

```bash
sudo systemctl status nginx
```

### Step 5.1 - Create a Server Block

Create a new Nginx configuration file for your application:

```bash
sudo nano /etc/nginx/sites-available/myapp
```

Add the following configuration, replacing `myapp.example.com` with your actual domain:

```nginx
server {
    listen 80;
    server_name myapp.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Save and close the file. Enable the configuration by creating a symbolic link:

```bash
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
```

Test the Nginx configuration for syntax errors:

```bash
sudo nginx -t
```

You should see `syntax is ok` and `test is successful`. Reload Nginx to apply the changes:

```bash
sudo nginx -s reload
```

At this point, your application is accessible over HTTP at `http://myapp.example.com`. In the next step, you will add HTTPS.

## Step 6 - Secure the Application with SSL

Use Certbot to obtain a free SSL certificate from Let's Encrypt and configure Nginx to serve your application over HTTPS automatically.

Install Certbot and the Nginx plugin:

```bash
sudo apt install -y certbot python3-certbot-nginx
```

Obtain and install the certificate for your domain:

```bash
sudo certbot --nginx -d myapp.example.com
```

Certbot will:
1. Verify that you own the domain
2. Obtain a certificate from Let's Encrypt
3. Automatically update your Nginx configuration to enable HTTPS and redirect HTTP to HTTPS

When prompted, enter your email address (used for renewal notices) and agree to the terms of service.

After Certbot completes, your Nginx configuration will be updated automatically. Verify the renewal process works correctly:

```bash
sudo certbot renew --dry-run
```

Certificates are valid for 90 days. Certbot installs a systemd timer that automatically renews them before they expire, so no manual action is needed.

## Step 7 - Configure the Firewall

Allow only the necessary ports through the firewall:

```bash
sudo apt install -y ufw
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
```

The `Nginx Full` profile allows both port 80 (HTTP) and port 443 (HTTPS). Verify the rules are active:

```bash
sudo ufw status
```

## Step 8 - Verify the Deployment

Open your browser and navigate to:

```
https://myapp.example.com
```

You should see:

```
Hello from my Node.js app on Hetzner!
```

You can also verify the PM2 process is running and check the application logs:

```bash
pm2 status
pm2 logs myapp
```

To simulate a crash and confirm PM2 restarts the application automatically, you can kill the Node.js process:

```bash
kill $(pgrep -f "node app.js")
```

Wait a moment and run `pm2 status` again — the application will be back online.

## Conclusion

You have successfully deployed a production-ready Node.js application on your Hetzner server. Your setup includes:

- **Node.js 22 LTS** installed via NodeSource
- **PM2** managing the application process, with automatic restarts on crash and on server reboot
- **Nginx** acting as a reverse proxy, forwarding traffic from ports 80 and 443 to the Node.js application
- **Let's Encrypt SSL certificate** installed via Certbot, with automatic renewal
- **UFW firewall** allowing only SSH and web traffic

From here, you can deploy your real application by replacing the sample `app.js` with your own code, use `pm2 reload myapp` for zero-downtime deployments, and use `pm2 logs myapp` to monitor your application in production.
