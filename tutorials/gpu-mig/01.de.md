## Einführung

Dieses Tutorial erklärt wie man [Multi-instance GPU (MIG)](https://docs.nvidia.com/datacenter/tesla/mig-user-guide/latest/introduction.html) gemäß der Voraussetzungen aus der [offiziellen MIG-Dokumentation](https://docs.nvidia.com/datacenter/tesla/mig-user-guide/latest/getting-started-with-mig.html#prerequisites) aktiviert.

**Voraussetzungen**

* Einen Server mit:
  * Ubuntu 24.04
  * Eine GPU die MIG unterstützt (siehe [offizielle Liste (EN)](https://docs.nvidia.com/datacenter/tesla/mig-user-guide/latest/supported-gpus.html))
  * Zugriff zum Root-Benutzer oder einem Benutzer mit sudo-Rechten
* Einen NVIDIA-Account für das [NVIDIA Display Mode Selector Tool](https://developer.nvidia.com/displaymodeselector)

## Schritt 1 - Voraussetzungen installieren

Du kannst den [offiziellen Konfigurator](https://developer.nvidia.com/cuda-downloads?target_os=Linux&target_arch=x86_64&Distribution=Ubuntu&target_version=24.04&target_type=deb_local) verwenden, um für dein System zu prüfen mit welchen Befehlen [das NVIDIA CUDA Toolkit heruntergeladen](https://docs.nvidia.com/cuda/cuda-installation-guide-linux/#download-the-nvidia-cuda-toolkit) und [das lokale Repository installiert](https://docs.nvidia.com/cuda/cuda-installation-guide-linux/#ubuntu) werden kann.

Folgend werden die Befehle erklärt, die auf Ubuntu 24.04 mit `x86_64` und deb (local) benötigt werden.

* Aktualisiere das System und installiere Linux Kernel Header
  ```bash
  sudo apt update
  sudo apt install linux-headers-generic
  ```

<br>

* Lade die APT-Pinning-Datei und das CUDA-Repository herunter
  ```bash
  wget -O cuda.pin https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-ubuntu2404.pin
  wget -O cuda.deb https://developer.download.nvidia.com/compute/cuda/13.2.1/local_installers/cuda-repo-ubuntu2404-13-2-local_13.2.1-595.58.03-1_amd64.deb
  
  sudo mv cuda.pin /etc/apt/preferences.d/cuda-repository-pin-600
  sudo cp /var/cuda-repo-ubuntu2404-13-2-local/cuda-*-keyring.gpg /usr/share/keyrings/
  sudo dpkg -i cuda.deb
  ```

<br>

* Installiere das [NVIDIA CUDA Toolkit](https://developer.nvidia.com/cuda/toolkit) und den  NVIDIA Driver open kernel module flavor:
  ```bash
  sudo apt-get update
  sudo apt-get -y install cuda-toolkit-13-2
  sudo apt-get install -y nvidia-open
  reboot
  ```
  Die Installation dauert einige Minuten.

<br>

* Nach dem Neustart, prüfe ob alles wie erwartet installiert wurde:
  ```bash
  nvidia-smi --query-gpu=driver_version --format=csv,noheader
  ```
  Es sollte die Driver-Version angezeigt werden.  
  Wenn ein Fehler angezeigt wird, kann es sein, dass der Driver nicht richtig geladen wurde.

## Step 2 - NVIDIA Display Mode Selector Tool einrichten

Vor dem nächsten Schritt muss die `Display Mode Selector Tool`-Datei von NVIDIA heruntergeladen werden. Dafür wird ein Account benötigt. Weitere Informationen findest du in der [offiziellen NVIDIA-Dokumentation](https://developer.nvidia.com/displaymodeselector).

Nachdem die ZIP-Datei heruntergeladen wurde (z.B. `NVIDIA-Display-Mode-Selector-Tool-1.72.0-July25.zip`) kopiere sie auf den Server und entpacke sie, wie folgend erklärt.

* Wenn die ZIP-Datei auf dem lokalen Gerät ist, kopiere diese auf den Server und verbinde dich anschließend mit dem Server:
  ```bash
  scp NVIDIA-Display-Mode-Selector-Tool-1.72.0-July25.zip user@<ip_address>:~/
  ssh user@<ip_address>
  ```
  Die ZIP-Datei sollte nun im Hauptverzeichnis liegen.

<br>

* Verschiebe die ZIP-Datei auf dem Server in einen neuen Ordner und entpacke die Datei:
  ```bash
  mkdir ~/displaymodeselector && cd ~/displaymodeselector
  mv ~/NVIDIA-Display-Mode-Selector-Tool-1.72.0-July25.zip ~/displaymodeselector
  unzip NVIDIA-Display-Mode-Selector-Tool-1.72.0-July25.zip
  ```
  Der entpackte Ordner sollte eine Datei namens `displaymodeselector` enthalten.
  Der Einfachheit halber kann die Datei direkt in das Verzeichnis `~/displaymodeselector` verschoben werden.

  ```bash
  cp "NVIDIA Display Mode Selector Tool - 1.72.0-July25/linux/x64/displaymodeselector" .
  ```

<br>

* Der Ordner sollte jetzt folgende Dateien enthalten:
  ```shell
  holu@example:~/displaymodeselector$ ls -al
  -rw-rw-r--  displaymodeselector
  drwxrwxr-x  __MACOSX
  drwx------  'NVIDIA Display Mode Selector Tool - 1.72.0-July25'
  -rwxrwxr-x  NVIDIA-Display-Mode-Selector-Tool-1.72.0-July25.zip
  ```

<br>

* Mache `displaymodeselector` ausführbar und prüfe die Version, um zu bestätigen, dass es funktioniert
  ```bash
  chmod +x displaymodeselector
  sudo ./displaymodeselector --version
  ```
  Im Output sollte etwas wie `NVIDIA Display Mode Selector Utility (Version #.##.#)` angegeben werden.

## Step 3 - GPU-Modus ändern

Für MIG muss der Display-Modus auf `compute` gesetzt werden.

* Zeige den aktuellen Display-Modus an
  ```bash
  nvidia-smi -i 0 --query-gpu=pci.bus_id,mig.mode.current --format=csv
  ```
  Im Output wird vermutlich etwas wie `[N/A]` angegeben.

<br>

* Setze den Display-Modus auf "compute" und führe einen Neustart durch:
  ```bash
  sudo ./displaymodeselector --gpumode compute --auto
  sudo reboot
  ```

<br>

* Nach dem Neustart, zeige den neuen Display-Modus an:
  ```bash
  nvidia-smi -i 0 --query-gpu=pci.bus_id,mig.mode.current --format=csv
  ```
  Im Output sollte nun `Disabled` angegeben werden.

## Step 4 - MIG aktivieren

Nun da die Voraussetzungen erfüllt sind, kann Multi-Instance GPU (MIG) aktiviert werden.

* Prüfe den GPU-Status vor der Änderung:
  ```bash
  nvidia-smi
  ```

<br>

* Aktiviere MIG:
  ```bash
  nvidia-smi -i 0 -mig 1
  ```
  Der Output sollte `Enabled MIG Mode` enthalten.

<br>

* Prüfe den GPU-Status, die Kernel-Version und die GPU VBIOS-Version:
  ```bash
  nvidia-smi
  uname -r
  nvidia-smi --query-gpu=vbios_version --format=csv
  ```
  Der Output sullte "MIG devices" enthalten.

## Ergebnis

Multi-Instance GPU (MIG) sollte nun auf dem System aktiviert sein und du kannst wie in der offiziellen Dokumentation erklärt GPU-Instanzen erstellen und verwalten:

* [List GPU Instance Profiles](https://docs.nvidia.com/datacenter/tesla/mig-user-guide/latest/getting-started-with-mig.html#list-gpu-instance-profiles)
* [Creating GPU Instances](https://docs.nvidia.com/datacenter/tesla/mig-user-guide/latest/getting-started-with-mig.html#creating-gpu-instances)
